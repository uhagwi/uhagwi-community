🔴 즉시 (서비스가 안 돌아감)

U-1 도메인 uhagwi.com 연결 (선택) ⏱ 30~60분
현재 서비스는 uhagwi-community.vercel.app으로만 접근 가능. uhagwi.com은 미등록 → 사용자가 친구·클라이언트에게 공유할 때 신뢰도 떨어짐. 도메인 사면 Vercel에서 5분 내 연결 가능.
  1. 도메인 구매 — 가비아·후이즈·Cloudflare Registrar 중 택1 (연 1.5~3만원)
  2. Vercel 대시보드 → uhagwi-community 프로젝트 → Settings → Domains
  3. uhagwi.com 입력 → Add 클릭
  4. Vercel이 알려주는 A 레코드(76.76.21.21) 또는 CNAME(cname.vercel-dns.com)을 도메인 등록기관 DNS 설정에 입력
  5. 5~30분 대기 후 https://uhagwi.com 접속 확인
  6. 환경변수 NEXTAUTH_URL을 https://uhagwi.com으로 변경
⚠️ 도메인 변경 후 Discord OAuth Redirect URL도 같이 갱신 (U-2 참고).
U-2a Discord OAuth 앱 등록 (개발자·AI 빌더용) ⏱ 15분
Discord = 30% AI 빌더 타깃 채널. Discord 봇과 같이 동작하므로 우하귀 핵심 인프라. 일반 사용자에겐 진입 장벽 있어 U-2b Google과 같이 박음.
  1. Discord Developer Portal → New Application → 이름 "우하귀"
  2. 좌측 OAuth2 → General
  3. Redirects에 2줄 추가:
    • https://uhagwi-community.vercel.app/api/auth/callback/discord
    • http://localhost:3000/api/auth/callback/discord
  4. Save Changes
  5. Client ID 복사 → Vercel 환경변수 DISCORD_CLIENT_ID
  6. Reset Secret → Client Secret 복사 → DISCORD_CLIENT_SECRET
📍 OAuth Scopes: identify, email 체크. guilds는 봇 합류 단계(U-6).
U-2b Google OAuth 클라이언트 발급 (일반 사용자용) ⏱ 15분
Google = 70% 일반 사용자 타깃 (꽃집 사장·교사·마케터). 한국에서 가장 보편적인 SSO. Discord 계정 없는 사용자도 30초 안에 진입.
  1. Google Cloud Console → Select a project → New Project → 이름 "uhagwi"
  2. 좌측 메뉴 → APIs & Services → OAuth consent screen
    • User Type: External
    • App name: 우하귀, User support email: 본인 이메일
    • Scopes: openid, email, profile 추가
    • Test users 단계 통과 → publish app (검수 받지 않으면 외부 사용자 100명 제한 — 베타엔 충분)
  3. APIs & Services → Credentials → + Create Credentials → OAuth client ID
    • Application type: Web application
    • Name: "우하귀 Web"
    • Authorized JavaScript origins:
      • https://uhagwi-community.vercel.app
      • http://localhost:3000
    • Authorized redirect URIs:
      • https://uhagwi-community.vercel.app/api/auth/callback/google
      • http://localhost:3000/api/auth/callback/google
  4. Create 클릭 → 팝업의 Client ID 복사 → Vercel GOOGLE_CLIENT_ID
  5. Client secret 복사 → Vercel GOOGLE_CLIENT_SECRET
⚠️ 도메인 uhagwi.com 연결(U-1) 후엔 Authorized origins·redirect URIs 두 곳 다 갱신 필요.
U-2c NextAuth 공통 환경변수 + 첫 Redeploy ⏱ 5분
U-2a·U-2b로 클라이언트 4개(DISCORD_*·GOOGLE_*) 박혔으니 NextAuth 자체 시크릿·URL 박고 한 번 재배포해야 환경변수 반영.
  1. 로컬 터미널에서 openssl rand -base64 32 실행 → 출력 복사
  2. Vercel Settings → Environment Variables에 다음 추가:
    키값
    NEXTAUTH_SECRET방금 생성한 base64 32자
    NEXTAUTH_URLhttps://uhagwi-community.vercel.app (또는 도메인 연결 후 https://uhagwi.com)
  3. Deployments → 최신 배포 ⋯ → Redeploy (env 반영)
  4. /login 접속 → Google·Discord 두 버튼 표시 확인 → 각각 클릭 후 정상 콜백 확인
U-3 Supabase 프로젝트 생성 + 마이그레이션 적용 ⏱ 25분
DB 미연결 → publish·comments·reactions·harnesses 라우트가 모두 500 반환. localStorage만으로는 갤러리 공개·다른 사용자 카드 못 봄.
  1. Supabase Dashboard → New Project
  2. 이름 uhagwi, 리전 Northeast Asia (Seoul), 비밀번호 별도 저장
  3. 생성 완료 후 Settings → API → 다음 값을 Vercel 환경변수에 박기:
    키Vercel 환경변수명
    Project URLNEXT_PUBLIC_SUPABASE_URL
    anon publicNEXT_PUBLIC_SUPABASE_ANON_KEY
    service_roleSUPABASE_SERVICE_ROLE_KEY ⚠️ 절대 클라이언트에 노출 금지
  4. SQL Editor → New query
  5. 로컬 uhagwi/packages/db/supabase/migrations/ 폴더의 SQL 파일을 번호 순서로 복사·실행 (0001 → 0103 → 0104 → 0105_google_oauth.sql)
  6. 또는 CLI 사용: supabase link --project-ref <ref> + supabase db push
  7. Authentication → Policies에서 RLS 활성 여부 확인
⚠️ 신규 마이그레이션 2개:
  • 0103_phase2_manifest.sql — Phase 2 산출물 manifest
  • 0105_google_oauth.sql — users 테이블에 google_id 추가 + discord_id NOT NULL 완화. U-2b Google OAuth와 함께 반드시 적용 안 하면 Google 로그인 사용자 upsert 실패.
U-4 Anthropic API 키 박기 (인터뷰·분석) ⏱ 5분
키 없으면 /api/interview/chat·/api/interview/analyze 둘 다 동작 안 함 → 인터뷰 자체 불가.
  1. Anthropic Console → Create Key
  2. 키 복사 → Vercel 환경변수 ANTHROPIC_API_KEY
  3. Workspace 결제 카드 등록 (인터뷰 1회당 Haiku 4.5 ~$0.005, Opus 4.7 분석 ~$0.02)
💡 두타 하네스에 이미 키 있으면 재사용 가능 (요약본의 "두타 하네스에 있어 키 가서 찾아와" 메시지 참고).

🟡 권장 (며칠 내)

U-5 Upstash Redis 연결 (rate-limit 서버 측) ⏱ 10분
메모리 폴백은 박혀 있어 동작은 함. 다만 Vercel serverless는 인스턴스마다 메모리 분리 → 한 사용자가 다른 인스턴스로 라우팅되면 한도 우회 가능. 운영 가면 Upstash 필수.
  1. Upstash Console → Create Database → Redis
  2. 리전 ap-northeast-1 (Tokyo) · Eviction enabled
  3. 생성 후 REST API 탭의 URL·Token 복사 → Vercel 환경변수:
    • UPSTASH_REDIS_REST_URL
    • UPSTASH_REDIS_REST_TOKEN
  4. 무료 티어 1만 명령/일 — MVP 충분
U-6 Discord 봇 토큰 + Railway/Fly 배포 ⏱ 30분
웹은 살아 있지만 봇이 죽어 있음 → 디스코드 서버에서 명령 응답·게시 알림·신규 멤버 동기화 불가. apps/bot/이 별도 배포 필요(Vercel은 봇 안 돌림).
  1. Discord Developer Portal → Bot → Reset Token → 토큰 복사
  2. Privileged Gateway Intents에서 Server Members·Message Content 활성
  3. Railway 또는 Fly.io 가입
  4. Railway: New Project → Deploy from GitHub → uhagwi-community 선택 → Root Directory: apps/bot
  5. 환경변수 DISCORD_BOT_TOKEN·DISCORD_GUILD_ID·SUPABASE_* 박기
  6. Deploy → 로그에서 Logged in as ... 확인
  7. Discord 서버에 봇 초대: OAuth2 URL Generator → scopes bot applications.commands, permissions Send Messages, Read Message History, Manage Roles 체크 → URL 복사 후 브라우저 열기
U-7 초기 카탈로그 검수 (친밀 맥락 일반화 확인) ⏱ 10분
2026-05-05에 카드 4건 일반화했지만 사용자 본인이 보면 더 가릴 부분이 있을 수 있음 (빌더명 "이진명" 등).
  1. 관리자 가챠 → 📦 전체 18장 채우기 → 전체 표시
  2. 18장 한 장씩 훑으면서 본인이 노출 안 됐으면 하는 카드 메모
  3. 노출 원치 않으면 다음 turn에 "X·Y·Z 카드 더 일반화해줘" 또는 "빌더명 익명화해줘" 요청
